# Compliance Check — 合规检查工作流

当此技能被调用时，帮助用户进行合规自查和准备审计材料。

## 命令

- `/compliance-check gdpr` — GDPR 合规检查
- `/compliance-check soc2` — SOC 2 合规检查
- `/compliance-check all` — 全面合规检查

## GDPR 合规检查

### 步骤 1: 数据映射

```markdown
## 数据处理活动记录 (ROPA)

| 数据类型 | 处理目的 | 法律依据 | 存储期限 | 接收者 |
|---------|---------|---------|---------|--------|
| 客户姓名 | 账户管理 | 合同履行 | 5 年 | 内部 |
| 邮箱 | 营销 | 同意 | 直到撤销 | Mailchimp |
| IP 地址 | 安全 | 合法利益 | 90 天 | 日志服务 |
```

### 步骤 2: 合规检查清单

```markdown
## GDPR 合规检查清单

### 法律依据
- [ ] 有有效的隐私政策
- [ ] 获取明确的用户同意
- [ ] 记录同意时间和内容
- [ ] 提供撤回同意的机制

### 数据主体权利
- [ ] 访问权 — 提供数据导出
- [ ] 更正权 — 提供数据编辑
- [ ] 删除权 — 提供账户删除
- [ ] 便携权 — 提供数据迁移
- [ ] 限制处理权 — 提供暂停处理

### 数据安全
- [ ] 加密存储敏感数据
- [ ] 加密传输 (TLS)
- [ ] 访问控制 (RBAC)
- [ ] 审计日志
- [ ] 数据泄露通知流程 (72h)

### 数据处理协议
- [ ] 与所有处理者签署 DPA
- [ ] DPA 包含标准合同条款
```

## SOC 2 合规检查

### 信任原则检查

```markdown
## SOC 2 Type II 检查清单

### 安全性 (Common Criteria)
- [ ] 访问控制 — IAM, MFA
- [ ] 边界保护 — WAF, 防火墙
- [ ] 事件响应 — 事件处理流程
- [ ] 监控 — 日志、告警

### 可用性
- [ ] SLO 定义和监控
- [ ] 灾难恢复计划
- [ ] 容量规划
- [ ] 变更管理

### 处理完整性
- [ ] 数据验证
- [ ] 错误处理
- [ ] 审计日志

### 保密性
- [ ] 数据分类
- [ ] 加密
- [ ] 访问控制

### 隐私
- [ ] 隐私政策
- [ ] PII 处理流程
- [ ] 隐私影响评估
```

## 合规报告模板

```markdown
# 合规检查报告 — [日期]

## 检查范围
- 标准: [GDPR/SOC2/其他]
- 范围: [系统/部门/全公司]
- 检查日期: [日期]

## 检查结果

### ✅ 符合项
| 控制项 | 证据 | 状态 |
|--------|------|------|
| 访问控制 | IAM 配置截图 | ✅ 符合 |

### ⚠️ 部分符合项
| 控制项 | 差距 | 整改计划 |
|--------|------|---------|
| 日志保留 | 当前 30 天，需要 90 天 | 2024-01-15 前完成 |

### ❌ 不符合项
| 控制项 | 风险 | 紧急程度 |
|--------|------|---------|
| MFA 未启用 | 高 | 高 |

## 整改计划

| 项目 | 负责人 | 截止日期 | 状态 |
|------|--------|---------|------|
| 启用 MFA | @security | 2024-01-10 | 🔄 进行中 |
| 增加日志保留 | @sre | 2024-01-15 | ⏳ 待开始 |

## 下次检查日期
[日期]
```

## 输出

3. 整改计划
