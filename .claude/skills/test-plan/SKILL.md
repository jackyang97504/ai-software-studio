# Test Plan — 测试计划工作流

当此技能被调用时，帮助用户创建测试计划和测试用例。

## 工作流程

### 步骤 1: 理解测试范围

1. 读取功能 PRD (`docs/requirements/[feature]-prd.md`)
2. 读取技术设计 (`docs/design/[feature]-design.md`)
3. 确定测试范围和策略

### 步骤 2: 设计测试策略

```markdown
## 测试策略

### 测试金字塔
- 单元测试: 70% (大量、快速)
- 集成测试: 20% (中量、中速)
- E2E 测试: 10% (少量、慢速)

### 测试环境
- DEV: 本地开发
- STAGING: 预生产
- PROD: 生产

### 测试数据
- 匿名化生产数据副本
- 测试数据工厂 (Faker)
```

### 步骤 3: 创建测试用例

```markdown
## 测试用例

### TC-001: 用户登录成功

**优先级**: P0 (阻塞)
**类型**: 功能测试

**前置条件**:
- 用户已注册
- 用户状态为激活

**测试数据**:
- 邮箱: test@example.com
- 密码: Test123!

**测试步骤**:
1. 打开登录页面
2. 输入邮箱 test@example.com
3. 输入密码 Test123!
4. 点击登录按钮

**预期结果**:
- 登录成功
- 跳转到 /dashboard
- 显示用户欢迎消息

**实际结果**: _[执行后填写]_

**状态**: ✅ 通过 / ❌ 失败

---

### TC-002: 用户登录失败 — 错误密码

**优先级**: P0
**类型**: 功能测试

**前置条件**:
- 用户已注册

**测试步骤**:
1. 打开登录页面
2. 输入邮箱 test@example.com
3. 输入错误密码 WrongPassword
4. 点击登录按钮

**预期结果**:
- 显示错误消息 "邮箱或密码错误"
- 保持在登录页面
- 不创建会话
```

### 步骤 4: 自动化测试映射

```markdown
## 自动化测试映射

| 测试用例 | 自动化状态 | 测试框架 |
|---------|-----------|---------|
| TC-001 | ✅ 已自动化 | Playwright |
| TC-002 | ✅ 已自动化 | Playwright |
| TC-003 | 🔄 开发中 | Jest |
| TC-004 | ⏳ 待开发 | - |
```

## 输出

1. 测试计划文档
2. 测试用例列表
3. 自动化覆盖情况
4. 测试环境要求
