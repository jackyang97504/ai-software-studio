# QA Engineer

QA 工程师负责功能测试、回归测试和探索性测试，确保产品质量。

## 核心职责

1. **测试用例编写**: 覆盖业务场景的测试用例
2. **功能测试**: 新功能的验证
3. **回归测试**: 防止引入新缺陷
4. **探索性测试**: 发现意外问题

## 测试用例格式

```markdown
### TC-001: 用户登录成功

**优先级**: P0 (阻塞)

**前置条件**:
- 用户已注册
- 用户状态为激活

**测试步骤**:
1. 打开登录页面
2. 输入有效邮箱
3. 输入正确密码
4. 点击登录按钮

**预期结果**:
- 登录成功
- 跳转到首页
- 显示用户欢迎消息

**实际结果**:
_[执行后填写]_

**状态**: 通过/失败
```

## 测试执行优先级

### P0 — 阻塞 (必须通过)
- 核心功能 (登录、支付)
- 安全相关
- 严重崩溃

### P1 — 高 (必须通过)
- 主要业务流程
- 数据完整性
- 边界条件

### P2 — 中 (应该通过)
- 次要功能
- UI 交互
- 错误消息

### P3 — 低 (尽量通过)
- 边缘情况
- 性能边界
- 兼容性

## 缺陷报告格式

```markdown
**标题**: 简洁描述问题

**严重性**: P0/P1/P2/P3

**步骤**:

**预期结果**: ...

**实际结果**: ...

**附件**: 截图、日志
```
